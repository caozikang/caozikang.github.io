Windows加载NFS

2018-04-15

如何在 Windows Server 2012 上加载 Linux 上的 NFS 共享 跨系统的集成从来都不是一蹴而就的。看看这个步骤，可以省去N个小时。 ==以下操作在Windows Server端== 1.在控制面板， Turn Windows Feature On and Off 打开 Server Manager Features 里选择 Client for NFS，重启 OS。 2.运行Regedit，定位到分支： HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\ClientForNFS\CurrentVersion\Default 新建DWORD 32键值， 名为：AnonymousUID 值为：0 名为：AnonymousGID 值为：0 3.执行命令mount NFS报错“Network Error - 53”： mount -o mtype=soft retry=10 timeout=6 casesensitive=yes anon 172.31.4.144:/NFS/nfs W: C:\Users\Administrator>mount -o mtype=soft retry=10 timeout=6 casesensitive=yes anon 172.31.4.144:/NFS/nfs W: Network Error - 53 Type ‘NET HELPMSG 53’ for more information.

360云盘的秒传

2018-04-15

360云盘“复活”之后的定位还是比较明确的，用于商务和团队协作。 套餐价格 实测通过30M移动宽带上传到360云盘，1G的文件耗时30分钟，每秒600KBps。 从360云盘下载到阿里云的速度每秒2MBps左右，比较理想。 问题在于： 某些人上传的大文件是损坏的，但是360云盘的“秒传”一看，这个文件在360云端有啊，所以直接放上去了。 不让我上传新的覆盖。。。 解决方法：给大文件再加一层压缩，避免秒传识别。 希望360能改进，给用户一个提示说这个文件在云端找到并默认使用了秒传的版本，允许选择重新从本地上传。
